⚠️ P0 — HEVC/H.265 编码不兼容

视频使用 hvc1 (HEVC) 编码。Chrome/Firefox 不支持原生 HEVC 解码，仅 Safari (macOS/iOS) 和 Edge (Windows + 硬件解码) 支持。大多数浏览器需软解或完全无法播放，是卡顿的首要原因。

⚠️ P0 — 无自适应码率（ABR）

单一 1080p / 2943kbps 文件，所有用户无论网络条件都下载同一版本。弱网用户缓冲不足导致卡顿，强网用户也不必要地消耗带宽。

🔶 P1 — 无 CDN 分发

GitHub Pages 从单一源站提供服务，无全球 CDN 节点。中国用户访问 GitHub Pages 的延迟通常在 200-800ms，且带宽受限（约 1-5 MB/s），98MB 视频需 20-98 秒下载完毕。

🔶 P1 — moov atom 位置未确认

未使用 -movflags +faststart 编码。如果 moov atom（元数据）在文件末尾，浏览器必须下载整个文件才能开始播放，导致首帧延迟极高。

ℹ️ P2 — 码率偏高

2943 kbps 对于网页宣传视频偏高。YouTube 同等分辨率推荐 2500-4500 kbps，但企业官网宣传视频 1080p 通常 1500-2000 kbps 即可保证画质。

ℹ️ P2 — 缺少封面图（poster）

视频加载前显示深蓝色占位块，无实际视频封面图。用户无法预览内容，且首帧渲染前视觉体验差。

3视频编码优化 — H.264 + Web Faststart

3.1 转码为 H.264（解决浏览器兼容性）

将 HEVC 转为 H.264 (avc1)，使用 -profile:v high 确保最大兼容性，-movflags +faststart 将 moov atom 移至文件头部实现即时播放：

# 单文件 H.264 转码（快速方案，不使用 HLS） ffmpeg -i assets/about-intro.mp4 \ -c:v libx264 -profile:v high -level 4.0 \ -preset slow -crf 23 \ -pix_fmt yuv420p \ -c:a aac -b:a 128k -ar 44100 \ -movflags +faststart \ assets/about-intro-h264.mp4
参数值说明
-c:v libx264H.264 编码器替换 HEVC，所有浏览器兼容
-profile:v highHigh Profile最佳压缩效率，广泛兼容
-level 4.0Level 4.0支持 1080p@30fps，兼容 99% 设备
-preset slow慢速预设编码效率更高，文件更小（编码耗时换体积）
-crf 23质量因子 23视觉无损，码率约降低 40-60%
-pix_fmt yuv420p像素格式最大兼容性（避免 yuv444p 不兼容）
-movflags +faststart快速启动moov atom 移至文件头部，实现即时播放
预计效果：H.264 转码 + CRF 23 后，98MB → 约 35-45MB，码率从 2943 → 约 1000-1500 kbps。加上 faststart，首帧时间从"全量下载"降至 <1 秒。
4自适应码率流（HLS）实现方案

4.1 为什么选择 HLS？

方案自适应码率浏览器支持实现复杂度推荐
单文件 MP4 (当前)否100%低❌
HLS (M3U8 + TS)是Safari 原生 + 其他通过 hls.js中✅ 推荐
DASH (MPD)是通过 dash.js中高❌
WebRTC否现代浏览器高❌

4.2 FFmpeg 多码率 HLS 转码脚本

生成 4 个码率等级（360p/480p/720p/1080p），每段 10 秒的 TS 分片 + 主播放列表：

# ===== HLS 多码率转码脚本 ===== # 输入: assets/about-intro.mp4 (HEVC 1080p) # 输出: assets/hls/ 目录下的 M3U8 + TS 文件 ffmpeg -i assets/about-intro.mp4 \ -filter_complex "[0:v]split=4[v360][v480][v720][v1080]; \ [v360]scale=w=640:h=360[v360out]; \ [v480]scale=w=854:h=480[v480out]; \ [v720]scale=w=1280:h=720[v720out]; \ [v1080]scale=w=1920:h=1080[v1080out]" \ -map [v360out] -c:v:0 libx264 -profile:v:0 main \ -b:v:0 800k -maxrate:v:0 856k -bufsize:v:0 1200k \ -map [v480out] -c:v:1 libx264 -profile:v:1 main \ -b:v:1 1200k -maxrate:v:1 1284k -bufsize:v:1 1800k \ -map [v720out] -c:v:2 libx264 -profile:v:2 high \ -b:v:2 2000k -maxrate:v:2 2140k -bufsize:v:2 3000k \ -map [v1080out] -c:v:3 libx264 -profile:v:3 high \ -b:v:3 3000k -maxrate:v:3 3210k -bufsize:v:3 4500k \ -map a:0 -c:a:0 aac -b:a:0 96k \ -map a:0 -c:a:1 aac -b:a:1 96k \ -map a:0 -c:a:2 aac -b:a:2 128k \ -map a:0 -c:a:3 aac -b:a:3 128k \ -f hls \ -hls_time 10 \ -hls_playlist_type vod \ -hls_flags independent_segments \ -hls_segment_filename "assets/hls/v%v/seg_%03d.ts" \ -master_pl_name master.m3u8 \ -var_stream_map "v:0,a:0 v:1,a:1 v:2,a:2 v:3,a:3" \ "assets/hls/v%v/playlist.m3u8"

6前端播放器性能调优

6.1 HLS.js 集成方案

使用 hls.js 库实现浏览器端 HLS 自适应播放。Safari 原生支持 HLS，其他浏览器通过 hls.js MSE 实现：

// 检测浏览器 HLS 支持情况 function initVideoPlayer() { const video = document.querySelector('.about-video'); const hlsSrc = 'assets/hls/master.m3u8'; const mp4Fallback = 'assets/about-intro-h264.mp4'; // Safari 原生 HLS 支持 if (video.canPlayType('application/vnd.apple.mpegurl')) { video.src = hlsSrc; } // 其他浏览器使用 hls.js else if (window.Hls && Hls.isSupported()) { const hls = new Hls({ // ===== 自适应码率配置 ===== abrEwmaDefaultEstimate: 500000, // 初始带宽估计 500kbps（保守起播） abrEwmaLatencyDefaultEstimate: 2, // 延迟估计 startLevel: 0, // 从最低码率(360p)起播，快速首帧 capLevelToPlayerSize: true, // 不超过播放器显示尺寸 // ===== 缓冲策略 ===== maxBufferLength: 30, // 最大缓冲 30 秒 maxMaxBufferLength: 60, // 绝对上限 60 秒 backBufferLength: 10, // 后方保留 10 秒（减少内存） maxBufferSize: 60 * 1000 * 1000, // 60MB 内存上限 // ===== 分片加载 ===== fragLoadingTimeOut: 20000, // 分片加载超时 20s fragLoadingMaxRetry: 6, // 最多重试 6 次 fragLoadingRetryDelay: 500, // 重试间隔 500ms // ===== 错误恢复 ===== enableWorker: true, // Web Worker 解码（不阻塞主线程） lowLatencyMode: false, # 关闭低延迟模式（VOD 不需要） recoverMediaError: true, // 自动恢复媒体错误 }); hls.loadSource(hlsSrc); hls.attachMedia(video); // 监听码率切换事件（用于 UI 提示） hls.on(Hls.Events.LEVEL_SWITCHED, (event, data) => { const levels = ['360p', '480p', '720p', '1080p']; console.log(`码率切换至: ${levels[data.level]}`); }); } // 完全不支持 — 降级到 H.264 MP4 else { video.src = mp4Fallback; } }

6.2 关键性能优化点

优化项实现方式效果
低码率起播startLevel: 0 从 360p 起播首帧时间 <1s（仅需下载 1 个 10s 分片 ≈1MB）
Web Worker 解码enableWorker: true视频解码不阻塞 UI 主线程，滚动/动画流畅

播放器尺寸限制capLevelToPlayerSize: true小屏幕不请求 1080p，节省带宽

7.2 网络感知预加载

// 使用 Network Information API 检测网络质量 function getNetworkQuality() { const conn = navigator.connection || navigator.mozConnection || navigator.webkitConnection; if (!conn) return 'unknown'; const effectiveType = conn.effectiveType; // 'slow-2g' | '2g' | '3g' | '4g' const downlink = conn.downlink; // Mbps if (effectiveType === '4g' && downlink > 5) return 'fast'; if (effectiveType === '4g' || effectiveType === '3g') return 'medium'; return 'slow'; } // 根据网络质量调整 HLS.js 配置 function getHlsConfig() { const quality = getNetworkQuality(); const configs = { fast: { startLevel: 2, maxBufferLength: 30 }, # 720p 起播 medium: { startLevel: 1, maxBufferLength: 20 }, # 480p 起播 slow: { startLevel: 0, maxBufferLength: 15 }, # 360p 起播 unknown:{ startLevel: 0, maxBufferLength: 20 }, # 保守起播 }; return configs[quality]; }

7.3 Service Worker 缓存（可选增强）

// sw.js — Service Worker 缓存 HLS 分片 const CACHE_NAME = 'video-hls-v1'; const VIDEO_CACHE = 'video-segments-v1'; self.addEventListener('fetch', (event) => { const url = new URL(event.request.url); // TS 分片 — 缓存优先（永久缓存已下载的分片） if (url.pathname.endsWith('.ts')) { event.respondWith( caches.open(VIDEO_CACHE).then(cache => cache.match(event.request).then(cached => cached || fetch(event.request).then(response => { cache.put(event.request, response.clone()); return response; }) ) ) ); } // M3U8 播放列表 — 网络优先（需获取最新版本） if (url.pathname.endsWith('.m3u8')) { event.respondWith( fetch(event.request).then(response => { const clone = response.clone(); caches.open(CACHE_NAME).then(cache => cache.put(event.request, clone)); return response; }).catch(() => caches.match(event.request)) ); } });
